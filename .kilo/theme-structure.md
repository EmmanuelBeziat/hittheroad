# WordPress Theme Structure (`hittheroad-2021`)

The theme is a custom WooCommerce-enabled WordPress theme located at `www/wp-content/themes/hittheroad-2021`.

## Core Structure
```text
hittheroad-2021/
├── assets/             # Generated by Grunt (DO NOT EDIT MANUALLY)
│   ├── css/
│   ├── js/
│   ├── images/
│   └── fonts/
├── classes/            # Core PHP functionality organized by responsibility
├── inc/                # Additional functional files (e.g., disable-comments)
├── template-parts/     # Reusable template fragments (header, footer, content, products-list)
├── woocommerce/        # WooCommerce template overrides (kept up-to-date with WC versions)
├── functions.php       # Main theme entry point
├── header.php          # Standard header
├── footer.php          # Standard footer
├── page-home.php       # Custom home page template
├── page-movies.php     # Custom movies page template
└── page-charte.php     # Custom charter page template
```

## PHP Classes (`/classes`)
The theme uses an object-oriented approach for its core functionalities, instantiated directly in `functions.php`:

| File | Responsibility |
|------|----------------|
| `htr-acf.php` | Advanced Custom Fields integration and helpers. |
| `htr-admin.php` | WordPress admin panel customizations. |
| `htr-custom-post-types.php` | Registers custom post types (e.g., `movie`). |
| `htr-scripts.php` | Enqueues frontend and login CSS/JS assets. |
| `htr-templates.php` | Custom template hierarchy and page logic. |
| `htr-tools.php` | General utility functions and helpers. |
| `htr-walkers.php` | Custom menu walkers (e.g., `class-wp-bootstrap-navwalker.php`). |
| `htr-woocommerce.php` | Heavy WooCommerce customizations (variation fields, pack systems, shipping classes). |

## Key Customizations
1. **ACF on Product Variations**: The theme heavily customizes WooCommerce to allow ACF fields directly on product variations. Custom hooks (`woocommerce_product_after_variable_attributes`, `woocommerce_save_product_variation`) handle rendering and saving of these fields.
2. **Product Packs**: Custom logic in `htr-woocommerce.php` handles automatic promotional pricing and front-end display for product packs.
3. **WooCommerce Overrides**: The `/woocommerce` directory contains overridden templates. **CRITICAL**: These must be manually updated when WooCommerce releases new versions (tracked in `CHANGELOG.md`).
4. **Disabled Features**: Comments and WordPress default sitemaps are explicitly disabled for security and clutter reduction.

## Enqueue Logic
Assets are loaded via `HTR_Scripts`:
- `app.css` (main styles)
- `vendors.js`, `classes.js`, `app.js` (scripts, loaded in footer)
- Optional `custom.css` and `custom.js` (if they exist in the theme, allowing quick overrides without the build pipeline).
