# Project Context & Guidelines

For all project-specific instructions, architecture details, workflows, and coding conventions, **always refer to the markdown files located in the `.kilo/` directory**.

Key files to consult:
- `.kilo/project-overview.md` : General project information, tech stack, and development commands.
- `.kilo/markup-workflow.md` : Rules for the frontend asset pipeline (Grunt, Sass, JS) and the `/markup` directory.
- `.kilo/theme-structure.md` : WordPress theme architecture, PHP classes, and template hierarchy.
- `.kilo/plugins-and-integrations.md` : List of active plugins and external API integrations (e.g., Google Sheets).
- `.kilo/release-workflow.md` : Mandatory steps to follow when executing a project update or release (Changelog, package.json, style.css).

Always prioritize the guidelines defined in `.kilo/` over general assumptions.
